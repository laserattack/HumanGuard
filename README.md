# HumanGuard

## Требования

- *Бэкенд*                : Go, net/http
- *Формат решения*        : прокси сайт с настройкой (SaaS решение)
- *Фронтенд*              : Angular
- *БД*                    : PostgreSQL
- *Аутентификация*        : по email
- *Деплой*                : автоматическое развертывание и тестирование на стороне сервера
- *Панель администратора* : авторизация, статистика запросов к сайту, возможность включение/выключение подсистем реагирования, возможность изменения параметров для модулей, графики и тд
- *Процесс разработки*    : SSDLC

## Модуль сбора поведенческих данных

Сбор клиентских метрик на стороне браузера:
- *Движения мыши, клики, скроллинг* 
- *Время между действиями, скорость набора текста* 
- *Размер окна браузера, разрешение экрана* 
- *Список установленных шрифтов, плагинов (через fingerprint)* 
- *Передача данных на сервер через асинхронные запросы (Beacon API / fetch) с привязкой к сессии* 

## Модуль анализа паттернов обращений

Анализ HTTP-запросов на серверной стороне:
- *Частота запросов с одного IP, User-Agent* 
- *Временные паттерны (регулярность, паузы)* 
- *Последовательность переходов (referrer, путь)* 
- *headless-браузеры* 
- *Использование эвристик и правил для первичной фильтрации* 

## Модуль детекции аномалий
- *Получение исторических данных из БД*
- *Глубокий анализ подозрительных сессий, отфильтрованных модулем первичного анализа*
- *Присвоение каждой сессии «оценки риска» (0 – легитимный, 100 – явный бот)* 

## Модуль реагирования

Настраиваемые действия в зависимости от уровня риска:
        
- *Низкий риск: пропуск без дополнительных проверок*
- *Средний риск: показ CAPTCHA*
- *Высокий риск: блокировка доступа с сообщением, временная блокировка IP, добавление в чёрный список*
- *Возможность ручного вмешательства администратора (разблокировка, изменение статуса)*

## Модуль логирования и мониторинга

- *Запись всех событий (попытки доступа, результаты детекции, действия администратора) в базу данных*
- *Визуализация в админ-панели: графики активности, список подозрительных сессий, история IP*

## Модуль управления настройками

- *Админ должен настраивать пороги срабатывания*
- *Возможность включать/выключать подсистемы*

## Модуль аутентификации

  - *Регистрация по email с подтверждением*
  - *Восстановление пароля*
